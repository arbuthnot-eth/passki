Bulk query — paste into GraphiQL at api.indexer.xyz/graphql

# POST https://api.indexer.xyz/graphql
# Headers:
#   Content-Type: application/json
#   x-api-user: <your username>
#   x-api-key:  <your api key>

query SkiPortfolio($owners: [String!]!) {
  sui {
    nfts(
      where: {
        owner: { _in: $owners },
        collection: { slug: { _eq: "suins" } }     # SuiNS collection on indexer
        # alt: nft_type: { _eq: "0x...::suins_registration::SuinsRegistration" }
      }
      order_by: [{ name: asc }]
      limit: 500
    ) {
      id
      name
      owner
      version
      staked
      burned
      collection { id slug title floor }
      listings(where: { listed: { _eq: true } }, order_by: { price: asc }, limit: 1) {
        id price price_str market_name
      }
      bids(where: { status: { _eq: "active" } }, order_by: { price: desc }, limit: 1) {
        id price price_str bidder
      }
    }
    # Floor per-collection (single call, same chain)
    collections(where: { slug: { _eq: "suins" } }) {
      id slug title floor supply
    }
  }
}

API discovery (2026-04-17)

Endpoint:    https://api.indexer.xyz/graphql      (Hasura, one POST for all chains)
Auth:        x-api-user + x-api-key  (no Bearer, no anonymous tier)
Free tier:   Must request a key at tradeport.xyz/docs -> "Request API Key"
Schema:      root wraps chains: { aptos { ... }, sui { ... }, iota { ... } }
Tables:      nfts, collections, listings, bids, actions, wallet_stats,
             wallet_holdings, wallet_trades, crypto_rates
Bulk owners: YES, native Hasura filter:  where: { owner: { _in: [...] } }
Joinable:    YES, listings / bids / collection / actions are all nested relations
             (single POST returns names + listing + top offer + floor)
Live test:   api.indexer.xyz responded { errors:[{message:"Access to resource denied"}] }
             to anonymous/dummy headers -> key required, but endpoint is up.
Docs site:   docs.indexer.xyz returns Vercel 404 (DEPLOYMENT_NOT_FOUND).
             Use www.tradeport.xyz/docs/nft-data-api/* instead (mirror).

Bulk-renew feasibility

TradePort:   No bulk-renew endpoint. Trading SDK covers list/buy/bid only.
SuiNS SDK:   SuinsTransaction exposes .renew(...) per name. Sui PTBs allow many
             moveCalls in ONE Transaction -> call suinsTx.renew() per name
             against the same gas coin, single signature, single on-chain tx.
Gas trick:   build one tx, split gas once via tx.gas, pass to each renew().
             Pyth priceInfoObject is fetched ONCE with tx.gas and reused across
             every renew() call (see buildRegisterSplashNsTx pattern in src/suins.ts).
             Rumble-balance pricing (82.5% list) applies if paying in NS.
Ceiling:     Sui tx size cap ~128KB and ~1024 commands -> ~200+ renewals per tx
             is comfortably safe. Chunk to 50/tx for faster finality + clearer UX.
