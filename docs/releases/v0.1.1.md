# v0.1.1 — Klefki

**Codename:** Klefki (Transform + Reset + Lock On)
**Released:** 2026-04-19
**Type:** Rebrand release — the product is now `passki`
**Commits on the arc:** 4 (`Klefki Transform`, `Klefki Reset`, `Klefki Lock On`, CI Pokemon-BST bump)

The first release under the new name. Formerly `.SKI` / `sui.ski`, this release takes the product brand to **passki** — the npm package, the GitHub repo, the primary domain. The Cloudflare Worker stays named `dotski` for Durable Object state preservation; nothing internal moved.

Named Klefki because Klefki is the key-ring Pokemon — fitting for a rebrand where the product *is* the key.

GitHub auto-redirects every old URL (`github.com/arbuthnot-eth/.SKI/...`) permanently. The legacy `sui.ski` domain still routes to `dotski`, so existing SuiNS profile pages (`<name>.sui.ski`) and deep links keep working.

The old `sui.ski@4.5.1` stays on npm as a tombstone — not unpublished, not deprecated. Consumers migrate at their own pace by swapping `npm i sui.ski` for `npm i passki`.

### Fresh semver (Klefki Reset — `559b9f9`)

`passki` is a new package with its own version history. Starts at `v0.1.0` (tagged baseline), first publish lands at `v0.1.1` via the Pokemon-BST workflow. The old 4.x versioning is preserved on the `sui.ski` package as its own history.

### Security headers + WebAuthn best practices (Klefki Lock On — `dad7846`)

Three additions to the worker's response middleware (`src/server/index.ts`):

- **HSTS on every worker response.** `Strict-Transport-Security: max-age=63072000; includeSubDomains; preload` — a WebAuthn relying-party domain must never downgrade to HTTP. CF edge HSTS (6-month toggle) layers on top for zone-wide coverage including static-asset paths.
- **CSP `frame-ancestors` extended** to include `https://*.passki.xyz` alongside the existing `https://*.sui.ski`. Lets the widget embed in both brand surfaces.
- **`/.well-known/change-password`** endpoint returns `303 → /#settings` per the [W3C Credential Management change-password URL spec](https://w3c.github.io/webappsec-change-password-url/). Browser password managers and operating-system credential UIs route the user here when they ask to rotate a credential. Passki's answer: open settings, rotate your passkey.

The `wrangler.jsonc` `run_worker_first` list now includes `/.well-known/*` so the assets binding does not shadow these endpoints.

### Cloudflare routing (pre-Klefki foundation)

Two custom domains on the `dotski` worker (`wrangler.jsonc`):

```jsonc
{ "pattern": "passki.xyz", "zone_id": "...", "custom_domain": true },
{ "pattern": "www.passki.xyz", "zone_id": "...", "custom_domain": true }
```

`custom_domain: true` tells Cloudflare to auto-provision Universal SSL + DNS records — no manual A-record juggling. An Advanced Certificate covering `passki.xyz` + `*.passki.xyz` layered on top for the subdomain expansion coming in later arcs.

Zone-level toggles set on the dashboard for `passki.xyz`:

- **Always Use HTTPS** — ON (HTTP → HTTPS 301 at edge)
- **HSTS** — ON, 6-month max-age, `includeSubDomains: ON`, `preload: OFF`
- **Automatic HTTPS Rewrites** — ON
- **SSL/TLS encryption mode** — Full
- **No-Sniff Header** — ON

Preload stays off until passki is stable enough to never need a plaintext dev subdomain — once on the Chromium preload list, removal takes months.

## What did not change

- **Worker internals.** Every Durable Object (SessionAgent, UltronSigningAgent, ShadeExecutorAgent, IntentRegistry, Chronicom, Pokedex, all 19) keeps its name, its storage, its state. The worker named `dotski` in `wrangler.jsonc` is the namespace for DO state; renaming the worker would have orphaned every DO. Hard no.
- **Internal lore.** "SKI Pass," "Key-In," the Pokemon versioning scheme, the Terminology table ("Storm," "Thunder," "Quest," "Purge," "Rumble," "Whelm"), ultron-as-keeper, brando-as-origin — all canonical. SKI is still the in-project shorthand for the sign-in primitive.
- **First commandment.** IKA-native, keyless agents. The three private-key secrets on the worker (`ENS_SIGNER_PRIVATE_KEY`, `SHADE_KEEPER_PRIVATE_KEY`, `ULTRON_PRIVATE_KEY`) remain an open migration — tracked in the IKA-native rollout, not this release.
- **Transport stack.** gRPC + GraphQL + JSON-RPC resilience layers are untouched.
- **Contracts.** Move packages, SUIAMI, Shade, Aegislash, Silvally — no on-chain changes.

## Consumer migration

For anyone depending on the npm package:

```bash
npm uninstall sui.ski
npm install passki
```

And in code:

```diff
- import { openModal, signIn } from 'sui.ski'
+ import { openModal, signIn } from 'passki'
```

The default export, named exports, custom elements, embed script, and styles are all byte-identical to `sui.ski@4.5.1`. Only the package name changed.
